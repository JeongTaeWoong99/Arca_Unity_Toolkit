# 스킬 동기화 (프로젝트 ↔ 전역 마스터)

작업 중 프로젝트에서 수정한 규칙을 전역 마스터에 반영하거나,
마스터의 최신 내용을 현재 프로젝트로 가져온다.

- 마스터 위치: `~/.claude/skills/unity-project-setup/templates/`
- 프로젝트 위치: `.claude/skills/` + 루트 `CLAUDE.md`

---

## 절차

### 1. Diff 수집

- 프로젝트 `.claude/skills/`의 각 SKILL.md를 마스터 `templates/skills/`의 같은 경로 파일과 비교한다.
- 프로젝트 `CLAUDE.md`를 마스터 `templates/CLAUDE.md.template`과 비교한다
  (placeholder·프로젝트 고유 내용은 차이로 치지 않는다).
- 한쪽에만 존재하는 스킬도 목록에 포함한다.
- 차이가 없으면 "동기화 완료 상태"라고 보고하고 종료한다.

### 2. 변경 건별 분류

각 차이를 아래 기준으로 **범용** / **프로젝트 특화**로 분류한다.

| 분류 | 기준 | 처리 |
|------|------|------|
| 범용 | 어떤 유니티 프로젝트에도 적용 가능한 규칙 (코딩 스타일, 설계 원칙, 커밋 규칙 등) | 마스터 반영 후보 |
| 프로젝트 특화 | 프로젝트 고유 경로·고유명사·서버 스펙·특정 에셋/씬 이름이 포함된 내용 | 프로젝트에만 유지 |

### 3. 사용자 확인 후 적용

- 분류 결과를 표로 보여주고, **범용 후보만** 마스터 반영 여부를 사용자에게 확인받는다.
- 승인된 항목을 방향에 맞게 적용한다:
  - **push**: 프로젝트 → 마스터 (`templates/`의 해당 파일 갱신)
  - **pull**: 마스터 → 프로젝트 (마스터가 더 최신일 때; 프로젝트 특화 수정을 덮어쓰지 않게 주의)
- 새 스킬이 프로젝트에서 만들어졌고 범용이면, 마스터에 추가하고
  `templates/CLAUDE.md.template`의 Skills 참조표에도 행을 추가한다.

### 4. 날짜 갱신 및 마무리

- **수정된 모든 md 파일**(마스터·프로젝트 양쪽)의 상단 `> 최종 업데이트:` 라인을
  오늘 날짜(YYYY-MM-DD)로 갱신한다. 수정하지 않은 파일의 날짜는 건드리지 않는다.
- 적용 결과를 요약 보고한다: 마스터 반영 N건 / 프로젝트 유지 N건 / pull N건.
- 마스터가 git 저장소면 커밋을 제안한다 (형식: `docs: 스킬 동기화 - <요약>`).

---

## 규칙

- 사용자 확인 없이 마스터를 수정하지 않는다.
- 마스터의 `> 최종 업데이트:` 날짜가 프로젝트 사본보다 최신이면, push 전에
  다른 프로젝트에서 먼저 반영된 변경이 있는지 diff를 보여주고 병합 방향을 확인한다.
